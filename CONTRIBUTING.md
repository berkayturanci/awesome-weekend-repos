# Contributing

This list isn't hand-edited — `README.md` and `index.html` are generated
from the same catalog that [@weekendrepos](https://youtube.com/@weekendrepos)'s
videos are built from. That means the fastest way to contribute is:

## Suggest a repo

[Open an issue](https://github.com/berkayturanci/awesome-weekend-repos/issues/new?template=suggest-a-repo.yml)
with the repo link and one sentence on the problem it solves. The bar for
a good suggestion:

- **Solves one real problem.** Not a framework, not a meta-collection —
  a tool you'd actually reach for.
- **Explainable in under a minute.** If it needs a diagram to understand
  what it does, it's not a fit for this format.
- **Has a license.** No license, no video, no listing.
- **Actively maintained.** A recent commit, not an abandoned experiment.

Not every suggestion becomes a video — but every good one gets a look.

## Found something wrong?

A stale star count, a broken link, a repo that's since been archived —
[open an issue](https://github.com/berkayturanci/awesome-weekend-repos/issues/new)
and it'll get fixed on the next regeneration.

## Why no direct PRs to README.md / index.html?

Both files are regenerated from source data on every update, so a manual
edit would just get overwritten. Issues are the actual queue.
